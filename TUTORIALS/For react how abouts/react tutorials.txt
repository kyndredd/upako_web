*./App.css this is a path now the ./ means "look at the current folder" the pathing is initial 
to the current file your in so if nasa src kana and andun ung App.css use ./App.css (go to this
current folder and find App.css)
Pero kung nasa src/components/navbar.jsx and you want to import the App.css then do ../App.css 
(go back once so nasa src/ kana and find App.css)

* Standard map
App.jsx          → navigation map
AuthProvider     → user identity, and also all of its components are all global means you can 
                   use it in any jsx files. It is use for authentication or user verification 
                   and use it for accessing website features if authentication met.
ProtectedRoute   → access rules
Layouts          → page structure
Pages            → UI screens
Services         → API calls
Hooks            → reusable logic

src/
 ├── app/
 │    └── App.jsx (routes only)
 │
 ├── context/
 │    └── AuthProvider.jsx (user login/auth)
 │
 ├── routes/
 │    └── ProtectedRoute.jsx (role protection)
 │
 ├── layouts/
 │    ├── PublicLayout.jsx (navbar)
 │    ├── DashboardLayout.jsx (sidebar)
 │
 ├── pages/
 │    ├── Home.jsx
 │    ├── Login.jsx
 │    ├── About.jsx
 │    ├── OwnerDashboard.jsx
 │    ├── TenantDashboard.jsx
 │
 ├── services/
 │    ├── api.js (fetch calls)
 │
 ├── hooks/
 │    ├── useTenants.js
 │    ├── useAuth.js
 │
 ├── components/
 │    ├── Navbar.jsx
 │    ├── Sidebar.jsx
